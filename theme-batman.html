CASE_01

Wrapped

My personal favorite

Full Case Study Lo-Fi Prototype Walkthrough Mid-fi Sample Interview Hi-Fi Mockup

The Inspiration

Screen time is one of those problems everyone knows is hurting them, but most people don’t change their behavior. Even when users like tracking screen time, they often don’t take action to reduce it.
Most solutions fall into two buckets:

    Hard Control: app blockers and time limits which are effective, but feels restrictive
    Gamification: apps like Forest which are engaging, but can still be rigid—and blocks even the apps you might actually need
We wanted something more flexible and personal especially for students who aren’t just “wasting time,” but are getting pulled into distractions while trying to study.

The Concept

Wrapped is a productivity app designed for students who struggle with habitual digital distractions and messy study routines
Instead of locking you out of apps, Wrapped focuses on three ideas:

    Personalized task-based nudges: Prompts that actually reference your current work
    For example: “You still have ____ to finish. Want to keep scrolling or wrap this up first?”
    Student-focused study support: A study session mode with customizable widgets (timers, audio, short breaks, etc.) so students don’t have to bounce between apps and accidentally spiral into distractions.
    Light rewards and motivational cues: The goal is encouragement and progress visibility rather than heavy points and leaderboards that become a distraction themselves.

The Approach

Through early interviews, surveys, and behavioral observation, we found that motivation was strongly tied to visible progress and feelings of accomplishment. These findings led us to prioritize customizable workspaces, goal-based reminders, and flexible study workflows over strict blocking features.

Multiple interaction directions were explored before converging on a hybrid approach that balanced familiarity with engagement. A key innovation was the nudge system, which references users’ actual tasks rather than generic alerts. This transforms interruptions into meaningful decision points, helping students reconnect with their goals in context.

I developed low and medium fidelity prototypes in Figma and conducted iterative usability testing using think-aloud sessions and structured walkthroughs, then ran a mixed-methods usability study with participants comparing different study modes. Participants completed structured tasks while providing both quantitative feedback and qualitative insights, enabling us to evaluate clarity, navigation, and engagement across workflows.

CASE_02

PortObello

Smart Port Logistics Management System

GitHub Repository

The Stack

Node.js, Express.js, OracleDB, SQL, HTML/CSS/JS

The Inspiration

Port operations involve many interconnected records which, in real workflows, often end up fragmented across spreadsheets or inconsistent systems. This can make it hard to answer questions like "Which ships are carrying the highest-value cargo?" or "Which trade agreements cover all partners?"

PortObello was built to provide a single source of truth with a normalized relational schema and an API that supports both operational CRUD and analytical querying.

The Backend

Designed and implemented an Oracle relational schema modeling 10 interconnected entities, including:

  • Subtype modeling (Country -> HomeCountry / ForeignCountry)
  • Composite keys (for example, warehouse location)
  • FK relationships and integrity rules across the domain

Built a Node.js + Express REST API integrated with Oracle using the oracledb driver and connection pooling for reliable database access.

Implemented endpoints for:

  • Schema initialization and seeding (drop/create/seed) for repeatable environments
  • Core CRUD operations (insert/update/delete) with relational consistency
  • Analytical SQL queries demonstrating joins, projections, aggregates, and a division-style query (for example, "home countries with trade agreements covering all partners")
  • Dynamic filtering endpoints for user-driven querying (for example, ship queries based on input constraints)

Structured the backend with clear separation of responsibilities (routing/controller/service-style organization), plus consistent error handling and environment-based configuration.

CASE_03

Marketplace Redesign

Research Based UI Overhaul

Full Research Report Mid-Fi Walkthough

The Inspiration

Second hand marketplaces are convenient, but the buying flow can feel stressful when search options are chaotic or overwhelming and it is hard to judge seller reliability. This project focused on reducing that friction by improving findability and trust signals in browse, evaluate and message flows.

The Concept

I designed a vertical, low-fidelity prototype which was iteratively improved into a medium-fidelity prototype that keeps the familiar layout while tightening the decision making loop with category-specific filters, a minimum listing description requirement, and prominent buyer/seller reviews (star rating + sentiment keywords + detailed review access).

To support faster browsing, a Quick View on hover feature was added and a Previously Viewed section for comparison, plus an optional horizontal scroll browsing pattern inspired by "racks" in a physical second-hand store.

The Approach

Through a combination of UX research, product design and implementation minded UI decisions, I set evaluation goals, prototyped the key interactions, then ran a mixed-methods usability study with 6 participants using moderated think-aloud tasks, task timing, and post-task interviews.

Results showed improved learnability (average task time of 2:36 down to 1:36, a 60-second decrease) and strong navigation sentiment (7.83/10). Trust features performed especially well with participants reporting it was easy to assess seller reliability due to the optimized review system while horizontal scrolling received mixed feedback and was flagged for iteration.

CASE_04

Evently

Hackathon Project

GitHub Repository Application Walkthough

The Stack

React, MongoDB/Mongoose, Chart.js, Node.js, Express.js, JWT

The Inspiration

Event planning often falls apart at the details stage with dietary restrictions, accessibility needs, and RSVP certainty being unpredictable. The common workarounds and generic form tools do not feel purposefully built for collecting and summarizing guest needs for hosts. Evently was created as a hackathon project, turned game changing alternative focused on inclusive, accommodation-first event planning.

The Concept

Evently is a full-stack event planner where users can sign up and log in, create events, and capture guest inputs (for example, diet category, accessibility needs, and RSVP) in a structured way. Hosts can then view a dashboard style event page that aggregates responses into charts (Bar, Doughnut, and PolarArea) to quickly understand what accommodations are needed.

The Approach

Built a MERN implementation with security and data visualization in mind:

  • Backend (Node/Express + MongoDB/Mongoose): environment-based configuration, CORS-enabled API, and user authentication with bcrypt password hashing, Joi validation (including password complexity), and JWT tokens (1-day expiry). The API exposes endpoints for signup (POST /users) and login (POST /auth), and runs on port 3001 in development.
  • Frontend (React): route-guarded navigation based on a stored auth token, form-based signup/login flows wired to the API, and a UI built with Bootstrap/React-Bootstrap plus React Modal for event creation UX. The event view uses Chart.js (via react-chartjs-2) and client-side parsing to transform response data into chart-ready datasets.
